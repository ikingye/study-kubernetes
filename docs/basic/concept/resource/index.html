    声明式 API #

    • 提交一个定义好的 API 对象来 “声明”
    • 启动一个对应的控制器进行调谐 / 编排

    控制器 #

    一个 Kubernetes 的控制器，实际上就是一个死循环：

    • 不断地获取 “实际状态”，
    • 然后与 “期望状态” 作对比，
    • 并以此为依据决定下一步的操作

    workload #

    Kubernetes divides workloads into different types. The most popular types supported by Kubernetes are:

    • Deployments
    • StatefulSets
    • DaemonSets
    • Jobs
    • CronJobs
